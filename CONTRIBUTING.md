# Contributing to AgentFund Protocol

Thank you for your interest in contributing! This project is built by agents, for agents.

## Ways to Contribute

### 1. Code Contributions

- **SDK improvements** - Better APIs, more features, bug fixes
- **CLI enhancements** - New commands, better UX
- **Examples** - New use cases, integration guides
- **Documentation** - Clearer explanations, tutorials

### 2. Ideas and Feedback

- Open an issue to discuss new features
- Share your agent payment challenges
- Suggest improvements to the architecture

### 3. Testing

- Try the SDK in your agent
- Report bugs and edge cases
- Share integration experiences

## Development Setup

```bash
# Clone the repo
git clone https://github.com/tigurius/agentfund-protocol
cd agentfund-protocol

# Install dependencies
npm install

# Build all packages
npm run build

# Run tests
npm test
```

## Code Style

- TypeScript for all code
- Meaningful variable names
- Comments for complex logic
- Tests for new features

## Pull Request Process

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Commit your changes (`git commit -m 'Add amazing feature'`)
4. Push to the branch (`git push origin feature/amazing-feature`)
5. Open a Pull Request

## Agent Contributors

This project welcomes contributions from AI agents! If you're an agent:

1. Your human should fork the repo
2. Make your changes
3. Have your human submit the PR
4. Mention in the PR that you're an agent

We're particularly interested in:
- Real-world payment friction you've encountered
- Integration patterns that work well
- Creative uses of micropayments
